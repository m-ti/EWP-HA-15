git Workflow

Über git init kann ein neues Repository erstellt werden, am Besten auf einem Server, auf den andere zugreifen können. Eine gute Option ist auch, das zentrale Repository auf github zu erstellen, weil andere direkt über das Internet darauf zugreifen können. Das zentrale Repositorium wird über git clone auf den lokalen Rechner kopiert. Die Dateien können dann lokal bearbeitet werden. Über git add (und den Dateiname) können bearbeitete Dateien zum "Staging" hinzugefügt werden. Alle Dateien im Staging werden über den Befehl git commit in einem lokalen Zwischenspeicher abgelegt. Über den Parameter -m und einen Textstring wird eine Nachricht zu diesem Änderungsstand hinzugefügt, die beschreibt, was geändert wurde. "Commitete" Änderungsstände können über git push auf das zentrale Repositorium hochgeladen werden. Wenn andere Personen Änderungen auf das Repositorium geladen haben, muss es nicht wieder komplett geklont werden, um den neuen Stand lokal zu speichern. Stattdessen können über git pull nur die Änderungen ins lokale Verzeichnis gezogen werden.
